# AmoCRM Deduplicator
Микросервис на FastAPI для автоматической склейки дублей контактов в AmoCRM.

## Зачем это нужно
Амо часто плодит дубликаты при входящих заявках. Этот скрипт ловит вебхуки, ищет оригинал контакта по номеру телефона или нику в Telegram и аккуратно переносит в него все новые данные, примечания и сделки.

## Особенности

- Поиск дублей по стандартному полю телефона и кастомному полю Telegram

- Автоматический перенос примечаний и перепривязка сделок к оригиналу

- Защита от гонки вебхуков (обработка ситуаций, когда амо шлет несколько запросов одновременно при одном сохранении)

- Обход ошибки 405 на базовых тарифах: если API не дает удалить контакт физически, скрипт делает софт-делит, переименовывая клона в "Дубль..."

## Стек
Python 3.11, FastAPI, Uvicorn, HTTPX

## Как запустить через Docker

Клонируем репозиторий:
```
git clone https://github.com/твой_ник/amocrm_deduplicator.git
cd amocrm_deduplicator
```

Создаем файл .env в корне проекта. Важно: боевой токен не пушить в репозиторий.
Пример .env:
```
AMO_DOMAIN=твой_субдомен.amocrm.ru
AMO_ACCESS_TOKEN=твой_длинный_токен_авторизации
TELEGRAM_FIELD_ID=1404413
```

Собираем и поднимаем контейнер:
```
docker build -t amo_dedup .
docker run -d -p 8000:8000 --env-file .env amo_dedup
```

## Запуск локально без Docker

Ставим зависимости:
```
pip install -r requirements.txt
```

Создаем .env (как описано выше)

Запускаем сервер:
```
uvicorn main:app --host 0.0.0.0 --port 8000
```

## Настройка хука в AmoCRM

Идем в Настройки -> Интеграции -> Webhooks.
Добавляем ссылку на поднятый сервер с эндпоинтом /webhook на конце (например: https://твой-домен.com/webhook).
